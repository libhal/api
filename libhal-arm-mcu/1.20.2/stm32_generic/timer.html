STM32 Generic Timer APIs#

hal::stm32_generic::timer#

Defined in namespace hal::stm32_generic

#include <libhal-arm-mcu/stm32_generic/timer.hpp>

class timer#

Implements shared timer setup and control logic common to all STM32 series.

This class provides the common functionality for timer configuration and control, abstracting the parts of the timer interface that remain consistent across different STM32 series. It is intended to be used by series-specific implementations, which handle MCU-specific configuration details and pass any required parameters to these generic functions.

Public Functions

timer(hal::unsafe)#

Construct timer uninitialized.

The purpose of this is to send the settings through the initialize function instead, because all the settings are series-specific. Therefore in the series-specific implementations of the timer, the address is deduced from the timer, as well as all the interrupt configuration is done, then they are passed to initialize.

If this constructor is used, it is unsafe to call any API of this class before calling the initialize() API with the correct inputs. Once that API has been called without failure, then the other APIs will become available.

bool is_running()#

Determine if the timer is currently running.

Returns:

true - if a callback has been scheduled and has not been invoked yet, false otherwise.

void cancel()#

Stops a scheduled event from happening.

Does nothing if the timer is not currently running.

Note that there must be sufficient time between the this call finishing and the scheduled event‚Äôs termination. If this call is too close to when the schedule event expires, this function may not complete before the timer interrupt is triggered.

void schedule(hal::time_duration p_delay, u32 p_timer_clock_frequency)#

Schedule an interrupt to occur for this timer.

If this is called and the timer has already scheduled an event (in other words, is_running() returns true), then the previous scheduled event will be canceled and the new scheduled event will be started.

If the delay time result in a tick period of 0, then the timer will execute after 1 tick period. For example, if the tick period is 1ms and the requested time delay is 500us, then the event will be scheduled for 1ms.

If the tick period is 1ms and the requested time is 2.5ms then the event will be scheduled after 2 tick periods or in 2ms.

Parameters:

  • p_delay ‚Äì - the amount of time until the timer expires

  • p_timer_clock_frequency ‚Äì - the clock driving the timer

Throws:

hal::argument_out_of_domain ‚Äì - if p_delay cannot be achieved.

void initialize(hal::unsafe, void *p_peripheral_address, void (*initialize_interrupts_function)(), cortex_m::irq_t p_irq, cortex_m::interrupt_pointer p_handler)#

Initialize the timer with the series-specific settings.

This is where the constructed uninitialized timer gets initialized. It gets all the series-specific settings passed to it that it needs, and it handles them appropriately.

Parameters:

  • p_peripheral_address ‚Äì - the address of the chosen timer peripheral

  • initialize_interrupts_function ‚Äì - the function needed to initialize interrupts for the specific series stm32. For example, for the stm32f1 series, the function passed would be hal::stm32f1::initialize_interrupts().

  • p_irq ‚Äì - the irq number for the chosen timer

  • p_handler ‚Äì - the platform specific interrupt handler to be installed

Throws:

hal::device_or_resource_busy ‚Äì - if the timer interrupt vector is already in use.
